Labs

Lab 1: The Linux Shell (slides)

This assignment aims at making sure you familiarize yourself with basic C fundamentals and building simple C programs as well as basic Git commands and working through Github classroom. In this lab, we expect you to implement some basic commands using the repl.it Linux environment. Please attend the lab session with your laptop, and note that the lab assignment will be due by the end of the lab session.

The following links will help you to prepare for this lab session:

  • MIT Missing Semester The Shell
  • Stanford CS107 Unix videos 1-15, 24, 25

Lab 2: Bits, Ints and Floats, Vim (slides)

In this lab, we are going to discuss file permission in Linux, vi text editor, and bit representations of integers and floating point numbers. The lab practice will cover bitwise operations and floating point representations.

The following links will help you to prepare for this lab session:

  • MIT Missing Semester Editors (Vim)
  • Stanford CS107 Unix videos 28
  • Examples of bit manipulation problems and their solutions on COMP201-Fall-2020

Lab 3: C-Strings and Valgrind (slides)

In this lab, we are going to discuss strings in C, and also we will introduce you to Valgrind.

The following links will help you to prepare for this lab session:

  • Stanford CS107 Unix videos 26
  • Here are some exercises on strings at w3resource.com

Lab 4: Arrays/Pointer and GDB (slides)

In this lab, we are going to talk about GNU Debugger (gdb).

The following links will help you to prepare for this lab session:

  • Harvard CS50 short on GDB
  • GDB Cheatsheet

Lab 5: Structs and Working with multiple files, writing your own Makefiles (slides)

In this lab, we are going to talk about const, structs, make and Makefiles.

The following links will help you to prepare for this lab session:

  • Compiling and Linking, Alex Allain
  • Multi-file programs, Geoff Kuenning.
  • Tutorialspoint Makefile - Quick Guide
  • GNU Make Manual

Lab 6: Machine Programming with Assembly (slides)

In this lab, you will learn about machine programming with Assembly language.

The following links will help you to prepare for this lab session:

  • x86 Assembly Crash Course, HackUCF
  • CMU 15-213 x86-64 Machine-Level Programming
  • Stanford CS107 Guide to x86-64
  • Stanford CS107 x86-64 Reference Sheet

Lab 7: Runtime Stack (slides)

In this lab, you will learn about managing runtime stack with Assembly language.

The following links will help you to prepare for this lab session:

  • CMU 15-213 x86-64 Machine-Level Programming
  • Stanford CS107 Guide to x86-64
  • Stanford CS107 x86-64 Reference Sheet

Lab 8: Memory Organization (slides)

In this lab, you will learn about how you can effectively make use of cache memories to write more efficient code.

The following links will help you to prepare for this lab session:

  • Cache Simulator
  • Stanford CS107 Guide to callgrind
  • Callgrind manual guide

Lab 9: Code Optimization (slides)

In this lab, you will learn about how you can optimize your code on your own as well the optimization strategies that GCC follow to do this for you.

Related links:

  • GNU GCC Guide to Options That Control Optimization
